# 📋 작업 흐름 규칙 (WORKFLOW)

> Claude Code가 프로젝트 작업 시 따라야 할 핵심 워크플로우

---

## 🔄 기본 작업 사이클

```
┌─────────────────────────────────────────────────────────┐
│  1. 세션 시작 → CLAUDE.md 읽기                          │
│  2. PROGRESS.md 확인 → 현재 작업 파악                   │
│  3. 태스크 시작 → 상태 🚀 변경                          │
│  4. 서브태스크 진행 → 완료 시 [x] 체크                  │
│  5. 태스크 완료 → 상태 ✅ 변경 + 진행률 업데이트        │
│  6. 다음 태스크로 이동                                  │
└─────────────────────────────────────────────────────────┘
```

---

## 📝 태스크 상태 표기

| 상태 | 아이콘 | 의미 |
|------|--------|------|
| 대기 | ⏳ | 아직 시작 안 함 |
| 진행중 | 🚀 | 현재 작업 중 |
| 리뷰 | 🔍 | 검토 필요 |
| 블로커 | 🔥 | 외부 의존성 대기 |
| 완료 | ✅ | 작업 완료 |

## 📊 PROGRESS.md 업데이트 규칙

### 태스크 시작 시
```markdown
# Before
- [ ] TASK-003: 로그인 기능 ⏳

# After
- [ ] TASK-003: 로그인 기능 🚀
```

### 서브태스크 완료 시
```markdown
# Before
- [ ] TASK-003: 로그인 기능 🚀
  - [x] 3-1: 모델 정의
  - [ ] 3-2: API 구현

# After (3-2 완료)

  - [x] 3-1: 모델 정의
  - [x] 3-2: API 구현

```

## 📁 태스크 문서 관리

### 태스크 시작 시 생성할 파일
```
docs/TASKS/TASK-XXX/
├── README.md       # 태스크 개요, 목표, 완료 조건
├── SUBTASKS.md     # 서브태스크 상세 (선택)
└── NOTES.md        # 작업 중 메모 (선택)
```

### README.md 필수 항목
```markdown
# TASK-XXX: [태스크 제목]

## 목표
[이 태스크가 완료되면 무엇이 가능한지]

## 완료 조건
- [ ] 조건 1
- [ ] 조건 2
- [ ] 테스트 통과

## 관련 파일
- `src/xxx/...`

## 메모
[작업 중 기록]
```

## ⚠️ 중요 원칙

### 1. 한 번에 하나의 태스크만
```
✅ TASK-003 완료 → TASK-004 시작
❌ TASK-003 진행 중 + TASK-004 동시 진행
```

### 2. 서브태스크는 작게 분할
```
✅ 3-1: User 모델 정의 (30분)
✅ 3-2: 로그인 API 구현 (1시간)
❌ 3-1: 전체 인증 시스템 구현 (하루)
```

### 3. 완료 전 검증 필수
```
태스크 완료 표시 전:
- [ ] 코드 린트 통과
- [ ] 테스트 통과 (있는 경우)
- [ ] 완료 조건 모두 충족
```

### 4. 막히면 기록하고 다음으로
```
🔥 블로커 발생 시:
1. PROGRESS.md에 🔥 표시
2. 블로커 내용 기록
3. 가능한 다른 태스크 진행
```

## 📋 일일 작업 템플릿

```markdown
## 오늘 작업 (YYYY-MM-DD)

### 완료
- [x] TASK-002 완료
- [x] TASK-003 서브태스크 3-1, 3-2 완료

### 진행 중
- TASK-003: 3-3 테스트 작성 중

### 블로커
- 없음 (또는 구체적 내용)

### 내일 계획
- TASK-003 완료
- TASK-004 시작
```
